🗺️ The AI Landscape — A Quick Map

Artificial intelligence is everywhere — but the terminology can be confusing. Is machine learning the same as AI? What's the difference between a model and a product? This guide gives you a clear mental model to understand how it all fits together.

💡
TPM analogy: Think of AI like a Russian nesting doll. AI is the outer shell — the big umbrella. Inside it is Machine Learning. Inside ML is Deep Learning. And inside Deep Learning are Large Language Models (like ChatGPT). Each layer is more specific than the one surrounding it.

Two things people often confuse: Products vs. Models

📱

AI Products (Apps)

The interface you interact with. Examples: ChatGPT (chat app), Codex (coding agent), Claude.ai (chat app), Claude Code (coding agent).

⚙️

AI Models (Engines)

The underlying AI engine doing the work. Examples: GPT-4o, Codex-mini, Claude Sonnet, Claude Opus. Accessible via APIs.

🔑
Key insight: ChatGPT is to GPT-4 as a car is to its engine. You drive the car (use the product), but the engine (model) is what actually makes it go. Developers can access the engine directly via APIs to build their own products.

Key Takeaways

  • ✅ AI → ML → Deep Learning → LLMs: each is a subset of the previous
  • ✅ Products (ChatGPT, Claude Code) are applications built on top of models
  • ✅ Models (GPT-4, Claude Sonnet) are the AI engines accessible via APIs
  • ✅ The same model can power multiple products (e.g., Claude Sonnet powers Claude Code)

📊 What is Machine Learning?

Machine Learning (ML) is a way of building software where, instead of writing rules by hand, you give the system examples and let it figure out the patterns.

💡
TPM analogy: Imagine you want to build a spam filter. The traditional approach: write rules like "if email contains 'free money', it's spam." The ML approach: show the system 100,000 labeled emails (spam / not spam) and let it learn the patterns itself. The ML model will discover patterns you never thought to write.

Three types of Machine Learning

🏷️

Supervised Learning

Train on labeled examples. The model learns to predict outputs from inputs. (Email spam detection, image classification, price prediction)

🔍

Unsupervised Learning

Find hidden patterns in unlabeled data. The model groups similar things together. (Customer segmentation, anomaly detection)

🎮

Reinforcement Learning

Learn by trial and error with rewards and penalties. (Game-playing AI, robotics, recommendation systems)

ML in the real world — examples you know

🎬 Netflix recommendations
Supervised + collaborative filtering ML predicts what you'll want to watch next based on your history and similar users.
📧 Spam filters
Trained on millions of spam/not-spam examples. Updates continuously as spammers evolve their tactics.
🏦 Fraud detection
ML models flag unusual transactions in real time by learning patterns from historical fraud cases.
🔍 Search ranking
Google's ranking is heavily ML-driven — learning from billions of queries what results people actually find useful.

Key Takeaways

  • ✅ ML learns from data examples instead of hand-coded rules
  • ✅ Three types: Supervised (labeled), Unsupervised (unlabeled), Reinforcement (trial & error)
  • ✅ Data quality is the #1 factor in ML success
  • ✅ ML models need monitoring after deployment — data can drift over time

🧠 What is Deep Learning?

Deep Learning is a type of machine learning that uses artificial neural networks — systems loosely inspired by the human brain. The "deep" refers to the many layers in these networks.

💡
TPM analogy: Imagine a photo recognition system. A traditional ML model might need you to manually extract features like "has four legs" or "has fur." A deep learning model learns to extract its own features automatically — starting with edges and shapes in early layers, then assembling those into ears and eyes, then into "cat." No manual feature engineering required.

How a Neural Network works

Why Deep Learning is a breakthrough

🔧

Traditional ML

You must manually define which features to extract from data. Requires significant domain expertise.

🧠

Deep Learning

Learns to extract relevant features automatically. Scales dramatically better with more data.

Key Deep Learning architectures (you'll hear these terms)

🔄 CNN (Convolutional Neural Network)
Specialized for images. Learns spatial patterns. Powers face recognition, medical imaging, self-driving vision.
📝 RNN / LSTM
Designed for sequences (text, time series). Early approach to language tasks — mostly replaced by Transformers.
🔀 Transformer
The architecture behind all modern LLMs. Uses "attention" to understand context across long sequences. Introduced in 2017 ("Attention Is All You Need").
🎨 Diffusion Model
Powers image generation (DALL-E, Stable Diffusion, Midjourney). Learns to progressively "denoise" random noise into images.

Key Takeaways

  • ✅ Deep learning = ML with many-layer neural networks
  • ✅ "Deep" = many layers, not depth of understanding
  • ✅ Learns features automatically — no manual feature engineering
  • ✅ The Transformer architecture (2017) is what powers all modern LLMs
  • ✅ Needs lots of data and compute — but scales extremely well

💬 Large Language Models (LLMs)

A Large Language Model (LLM) is a deep learning model trained on enormous amounts of text. It learns to understand and generate language by predicting the next word (or "token") in a sequence — over and over, at massive scale.

💡
TPM analogy: Imagine reading every book, article, website, and code repository ever written — billions of pages. After reading all of that, you develop an incredibly rich intuition for language patterns, facts, reasoning, and writing styles. That's what an LLM has done, except it "read" everything in a compressed mathematical form.

What are "tokens"?

LLMs don't process whole words — they work with tokens, which are roughly word-pieces. Here's how the sentence "The bank by the river was flooded" gets split:

What LLMs can (and cannot) do

✅

LLMs are great at

Writing & editing · Summarization · Q&A · Code generation · Translation · Brainstorming · Following instructions · Conversation

⚠️

LLMs struggle with

Precise math · Reliable factual recall (can "hallucinate") · Real-time information · Counting · Multi-step logical chains

Key LLM concepts you'll hear

📐 Parameters / Weights
The learned numbers inside the model. GPT-4 has hundreds of billions of parameters. More parameters ≠ always better, but generally more capable.
🎯 Prompt Engineering
The art of crafting inputs to get the best outputs from an LLM. Clear, specific prompts with context produce dramatically better results.
🔧 Fine-tuning
Taking a pre-trained base model and training it further on a specific dataset to specialize it (e.g., medical Q&A, customer service).
📚 RAG (Retrieval-Augmented Generation)
Combining an LLM with a search system that retrieves relevant documents. Solves the "outdated knowledge" problem by injecting fresh context.
🧲 Context Window
How much text an LLM can "see" at once (input + output). GPT-4 supports 128K tokens; Claude 3 supports up to 200K tokens (~150,000 words).
🤖 AI Agents
LLMs that can take actions: browse the web, run code, call APIs, manage files. Coding agents like Codex and Claude Code are examples.

Key Takeaways

  • ✅ LLMs are deep learning models trained on massive text datasets
  • ✅ They work by predicting the next token — this scales to powerful capabilities
  • ✅ Attention mechanism lets them understand context across long sequences
  • ✅ Strong at language tasks; can "hallucinate" — always verify factual claims
  • ✅ Context window = how much text the model can process at once

🤝 AI-Assisted Development

AI is not just changing what we build — it is fundamentally changing how we build it. AI coding tools are now embedded directly in the software development lifecycle, acting as pair programmers, code reviewers, and autonomous agents that can complete entire tasks end-to-end.

💡
TPM analogy: Think of AI coding tools like a highly skilled contractor who never sleeps. You (or a developer on your team) can describe a task in plain language, hand it to the AI, and get back a pull request — complete with code, tests, and a description. Your job as a TPM shifts from coordinating tasks to reviewing outcomes.

How AI has changed the development lifecycle

🗓️

Traditional SDLC

  • • Developer writes code from scratch
  • • Manual code review for every PR
  • • Tickets assigned one-per-engineer
  • • Bottleneck: developer throughput
  • • Cycle time measured in days/weeks
🤖

AI-Assisted SDLC

  • • AI agents generate first-cut code
  • • AI assists with PR review & summaries
  • • One engineer can manage multiple AI agents
  • • Bottleneck shifts to review & judgment
  • • Well-scoped tasks can close in hours

🌀 Spectre Event Development

Spectre event development is an emerging AI-assisted development pattern where a team deliberately brings together developers, TPMs, and AI coding agents in a structured, time-boxed session to tackle a backlog of well-defined tasks collaboratively.

🔑
What makes a Spectre event: A Spectre event is a focused development sprint — typically a few hours to a day — where AI coding agents run in parallel on multiple issues simultaneously. Engineers act as reviewers and directors rather than primary code authors, dramatically multiplying the team's throughput.

What this means for TPMs

📐

Better issue quality

AI agents need crisp, well-scoped specs. Your ability to write clear acceptance criteria directly impacts how much the agent can do autonomously.

🔭

Shift to review culture

More PRs get opened faster. Teams need lightweight review processes — TPMs help design review criteria and escalation paths.

📊

New velocity metrics

Traditional story points may not capture AI-augmented throughput. TPMs must rethink how they measure and communicate team velocity.

Key Takeaways

  • ✅ AI coding tools act as autonomous agents, not just autocomplete — they open PRs and run code
  • ✅ The development bottleneck shifts from writing code to reviewing and directing AI
  • ✅ Spectre events are structured AI-assisted sprints where agents work in parallel on many issues
  • ✅ TPMs add value by writing crisp specs, designing review workflows, and rethinking velocity metrics
  • ✅ Well-scoped issues are the most important input to successful AI-assisted development
